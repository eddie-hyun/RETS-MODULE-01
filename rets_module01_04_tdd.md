## 1. 테스트 설계 개요

### 1.1 테스트 목적

본 테스트 설계서는 RETS-MODULE-01의 SRS에 정의된 기능 요구사항(FUNC-001~062), 비기능 요구사항(NFR-P01~M03) 및 유스케이스(UC-001~007)에 대한 검증 가능한 테스트 케이스를 정의한다. TDD(Test-Driven Development) 방법론에 따라 구현 전 테스트 케이스를 선 도출하며, 각 테스트 케이스는 Given-When-Then 형식으로 기술한다.

### 1.2 테스트 범위

| 구분 | 포함 | 제외 |
|------|------|------|
| 기능 테스트 | FUNC-001~062 전체 | 백엔드 서버 테스트 |
| 비기능 테스트 | 성능·사용성·보안·호환성 핵심 항목 | 부하 테스트(Load Test) |
| 통합 테스트 | LLM 릴레이 서버 연동, 파일 I/O | 타 모듈과의 연동 테스트 |
| 회귀 테스트 | 핵심 유저 시나리오 4종 | 전체 회귀 |
| 자동화 테스트 | 핵심 유저 시나리오 (Playwright) | 모든 TC 자동화 |

### 1.3 테스트 케이스 명명 규칙

```
TC-[순번 3자리]
예: TC-001, TC-025, TC-P01 (비기능), TC-U01 (사용성)
```

### 1.4 합격 기준 (Pass Criteria)

- **P1 (Must) 기능 TC**: 전체 Pass 율 100%
- **P2 (Should) 기능 TC**: 전체 Pass 율 90% 이상
- **P3 (Could) 기능 TC**: 전체 Pass 율 80% 이상
- **비기능 TC**: 목표값 기준 100% 충족
